Required CSV Format

Your CSV file must contain the following columns:

  • AssemblyCat - Assembly category (e.g., VFD, HS, WW)
  • Assembly_ID - Unique assembly identifier
  • Component_Part_Number - Part number for each component
  • Quantity - Quantity of each component
Optional columns: Assy_#, PanelAssy, ShopAssy (these will be ignored during import)
Example CSV Format
AssemblyCat Assembly_ID Component_Part_Number Quantity Assy_# PanelAssy ShopAssy
VFD PF755TS-005HP 20G21ND8P0JA0NNNNN 1 VFD001 1
VFD PF755TS-005HP JLS010 3 VFD001 1
VFD PF755TS-007.5HP 20G21ND011JA0NNNNN 1 VFD002 1
Import Process:
  1. Upload your CSV file using the form above
  2. Preview and edit assembly names and categories
  3. Select which assemblies to import
  4. Confirm import to create the assemblies
Note: If component parts don't exist in the parts database, placeholder parts will be created with $0.00 price. You can update these later in the Parts Database section.
Troubleshooting: If you get column errors, ensure your CSV file:
  • Uses UTF-8 encoding (save as CSV UTF-8 in Excel)
  • Has exact column names: AssemblyCat, Assembly_ID, Component_Part_Number, Quantity
  • Doesn't have extra spaces or quotes around column names
